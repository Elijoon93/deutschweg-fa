# DeutschWeg X13.7 RC1 — راهنمای پذیرش واقعی دستگاه

این نسخه تا تکمیل تست‌های زیر **Final نیست**.

## روش اجرا

1. فایل‌های ZIP را در ریشه Repository جایگزین و Push کنید.
2. GitHub Pages را روی گوشی باز کنید.
3. Site Data را پاک نکنید.
4. وارد `بیشتر → نهایی‌سازی و پذیرش دستگاه` شوید.
5. هر آزمون را روی گوشی واقعی انجام دهید.
6. تیک «روی دستگاه واقعی آزمایش کردم» را فعال کنید.
7. نتیجه را PASS یا FAIL ثبت کنید.
8. گزارش JSON پذیرش را Export کنید.

## آزمون‌های الزامی

1. ورودی هفت‌مرحله‌ای بدون برش یا جابه‌جایی افقی
2. نوار پایین: خانه، برنامه من، یادگیری، پیشرفت، بیشتر
3. منوی زمینه‌ای یادگیری: نمای کلی، موضوع‌ها، واژگان، تمرین، جست‌وجو
4. More Drawer: Search، Favorites، Recent و All Apps
5. بازشدن واقعی کتگوری‌ها و موضوع‌ها
6. عملکرد Browser Back
7. عدم هم‌پوشانی کیبورد با فیلدها و Bottom Navigation
8. Portrait و Landscape بدون Overflow
9. نصب PWA و اجرای آفلاین
10. حفظ Profile، Planner، SRS و Progress پس از Update
11. Export Backup و Restore
12. نمایش جداگانه Verified / Project-curated / Candidate

## معیار Final

فقط وقتی این دو شرط برقرار باشد:
- تمام کنترل‌های خودکار PASS باشند.
- هر ۱۲ آزمون واقعی دستگاه PASS ثبت شده باشد.

وجود کد یا Static QA به‌تنهایی معادل پذیرش دستگاه نیست.
