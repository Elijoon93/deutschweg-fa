DeutschWeg FA v6 — راهنمای انتشار و حالت چندکاربره

۱) انتشار ساده و رایگان روی GitHub Pages
- همه فایل‌های این پوشه را در ریشه Repository عمومی آپلود کنید.
- Settings > Pages > Deploy from a branch > main / root
- بدون هیچ تنظیم دیگری، هر کاربر همان لینک را باز می‌کند و داده‌هایش در مرورگر همان دستگاه ذخیره می‌شود.

۲) حالت حساب کاربری و Cloud Sync (اختیاری)
- در Supabase یک Project بسازید.
- فایل supabase_schema.sql را در SQL Editor اجرا کنید.
- Authentication > Providers > Email را فعال نگه دارید.
- در config.js فقط Project URL و public publishable/anon key را وارد کنید.
- هرگز service_role key را در GitHub قرار ندهید.
- به لطف RLS، هر کاربر فقط رکورد user_state خودش را می‌خواند/می‌نویسد.

۳) یک لینک برای چند کاربر
- همه کاربران یک URL واحد GitHub Pages دارند.
- در حالت محلی، داده‌ها روی مرورگر هر دستگاه جداست.
- در حالت Supabase، هر فرد با ایمیل/رمز وارد می‌شود و داده‌اش بین گوشی و رایانه همگام می‌شود.

۵) درباره محتوای فیلم
خود برنامه میزبان فایل فیلم نیست و فقط لینک خارجی ارائه می‌دهد. رعایت قوانین پلتفرم و حقوق نشر بر عهده منتشرکننده محتواست.

۶) متدها
بخش «متدها» رویکردهای مدرس و روش‌های پژوهشی را از هم جدا می‌کند. کارت استاد مرتضی فرهمند صرفاً خلاصه‌ای از رویکرد عمومی قابل مشاهده در کانال ایشان است و به‌عنوان «متد علمی اختصاصی» معرفی نشده است.
